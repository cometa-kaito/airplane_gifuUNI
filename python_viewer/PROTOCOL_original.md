# シリアル送受信の仕様（`viewer_serialsend.py` / `paperplane05-YPR.py`）

両スクリプトとも **USB シリアル（pyserial）** を使います。デフォルトのボーレートは **115200** です（`viewer_serialsend.py` は `--baud` で変更可能。`paperplane05-YPR.py` はコード内の `Serial(...)` を編集）。

---

## 1. `viewer_serialsend.py`

### 1.1 PC → マイコン（送信）

- **形式:** 表示中の **matplotlib ウィンドウにフォーカスがある状態**でキーを押すと、**その 1 文字だけ**が UTF-8 で送られ、**末尾に改行（`\n`）** が付きます。
- **送信される文字（`event.key` が次に一致するとき）**

| 送信文字 | 備考 |
|----------|------|
| `a`, `m`, `p`, `d`, `i`, `x`, `y`, `z`, `g`, `l`, `r`, `e`, `h`, `s` | 小文字 |
| `A`, `M`, `P`, `D`, `I`, `X`, `Y`, `Z`, `G`, `L`, `R`, `E`, `C`, `S`, `H` | 大文字 |
| `c` | 小文字の `c` のみ（大文字 `C` は上表） |

画面に表示される対応関係の例（スクリプト内ヘルプと同じ趣旨）:

- Auto / Manual / Center: `A`, `M`, `c`
- Pitch PID: `p` `P`, `d` `D`, `i` `I`
- Roll PID: `x` `X`, `y` `Y`, `z` `Z`
- Trim: `l` `L`, `r` `R`, `e` `E`
- Goal: `g` `G`
- Save / Reset: `s` `S`, `h` `H`

マイコン側は **1 行 1 コマンド**として解釈できる実装（例: 行単位で読んで先頭 1 文字をコマンドにする）を想定しています。送信側が付ける改行は `\n` です。

### 1.2 マイコン → PC（受信・テレメトリ）

- **行の区切り:** `readline()` 相当（**`\n` まで**）。行末の `\r` は `strip()` で除去されます。
- **テレメトリとしてグラフに乗る行**は、次を**すべて**満たす必要があります。
  - **カンマ区切りちょうど 18 フィールド**（`split(",")` の要素数が 18）。
  - 各フィールドが **`float()` で解釈可能**な数値文字列。

**フィールド順（固定）**

| 順番 | 名前 | 意味（スクリプト上の用途） |
|:----:|------|---------------------------|
| 1 | `seq` | 通番（整数想定だが内部は float として保持） |
| 2 | `t_ms` | 時刻 [ms]（グラフの横軸の基準） |
| 3 | `dt_ms` | サンプル間隔 [ms] |
| 4–6 | `ax`, `ay`, `az` | 加速度など |
| 7–9 | `gx`, `gy`, `gz` | 角速度など |
| 10–12 | `roll`, `pitch`, `yaw` | 姿勢 [deg] 想定 |
| 13–15 | `s0`, `s1`, `s2` | サーボ等のスカラー |

**例（改行は `\n`）:**

```text
0,12345,10,0.0,0.0,9.8,0,0,0,1.2,-3.4,0.5,90,90,90
```

### 1.3 マイコン → PC（テレメトリ以外の行）

18 フィールドに一致しない行は **テレメトリとしては破棄されず**、コンソールに `[LINE] ...` として表示されます。また、先頭が次のいずれかの行は **画面上のステータス／情報テキスト**の更新に使われます。

| 先頭プレフィックス | 用途 |
|--------------------|------|
| `[MODE]` | タイトル付近のステータス表示 |
| `[PARAM]` `[SCHEMA]` `[EVENT]` `[INFO]` `[READY]` | ウィンドウ下部の情報表示 |

ログ用の人間可読メッセージやスキーマ行を流す場合は、この形式に合わせるとビューア上で読みやすくなります。

## 2. `paperplane05-YPR.py`

### 2.1 PC → マイコン

- このスクリプトから **シリアル送信は行いません**（受信のみ）。

### 2.2 マイコン → PC（受信）

- **読み方:** `Serial.read_until(size=250)`（pyserial の **デフォルト終端は `\n`**）により、**最大 250 バイト**または **改行まで**読みます。
- 読めたバイト列を **UTF-8 として `decode()`** し、その文字列を **カンマで分割**しています（実装は `re.split` だが、区切りはカンマ相当）。

### 2.3 必要なフィールド数と並び

**先頭から 6 つの数値**が必須です（インデックス 0〜5）。足りないと `float()` や配列参照で例外になります。

スクリプト内の代入は次のとおりです（`data1` はカンマ分割の配列）。

| `data1` のインデックス | 代入先 | 意味（表示・後段のオイラー角 [deg] として） |
|:------------------------:|--------|---------------------------------------------|
| 0 | `data[11]`（内部で roll として radian 変換） | **roll** |
| 1 | `data[10]` | **pitch** |
| 2 | `data[9]` | **yaw** |
| 3 | `data[0]` | **ax**（加速度ベクトルの X 成分としてベクトル表示に使用） |
| 4 | `data[1]` | **ay** |
| 5 | `data[2]` | **az** |

つまり **1 行あたりの並びは次の 6 つ**です。

```text
roll,pitch,yaw,ax,ay,az
```

**例:**

```text
1.0,-2.5,90.0,0.01,0.02,9.81
```

- 単位の前提: コードでは `data[9]`,`[10]`,`[11]` を **度（degree）** とみなし、`math.radians()` してクォータニオンに変換しています。加速度は **正規化なしの数値**がそのまま矢印の描画にスケール `0.01` で使われます。
- **6 つより後ろにカンマ区切りで値があっても**、現状のコードは **先頭 6 つ以外は読んでいません**。

### 2.4 `viewer_serialsend` との対応関係

| 項目 | `viewer_serialsend`（テレメトリ 1 行） | `paperplane05-YPR`（先頭 6 フィールド） |
|------|----------------------------------------|------------------------------------------|
| 姿勢 | フィールド 10–12 が `roll`, `pitch`, `yaw` | `roll`, `pitch`, `yaw` が先頭 3 つ |
| 加速度 | フィールド 4–6 が `ax`, `ay`, `az` | 同じ順の `ax`, `ay`, `az` が 4–6 番目 |

**互換を取るなら:** テレメトリ行を **`roll,pitch,yaw,ax,ay,az` の順で先頭に並べ、その後ろに必要なら残りのフィールドを続ける**形にすると、`paperplane` は先頭 6 つだけを使い、`viewer_serialsend` は **18 フィールドすべて**が揃った行だけをプロット対象にできます。  
（完全に同じ 1 行を両方で使うには、`viewer_serialsend` 側が要求する **18 フィールド行**を送る必要があります。）
